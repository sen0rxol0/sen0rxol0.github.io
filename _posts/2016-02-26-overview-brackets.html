Brackets is a open source text/code editor, founded by Adobe to provide Developers a lightweight and powerful editor. Brackets has a lot of features and tools that can enhance code development process, although web development is its core, it can be used for any development that involves code development.

This article will be focused on a variety of brackets features, available options, usage and thoughts.

Features

Brackets as some of the best features that a developer can which for in a text editor.

Live Preview :
This feature allows you to live edit your project, you can see your changes in the browser as you develop the code in real-time.
Extract :
The Extract is a extension for Brackets that can be used to pull informations from a design in a PSD document.
Quick Edit :
Quick Edit is a really helpful feature which helps you modify your CSS, inline, within a HTML document, lets you modify a CSS color value from a inline color picker and also as some fuctionality for your JavaScript.
Quick Docs :
As helpful as Quick Edit, this feature provides you inline reference for CSS properties.
JSLint :
Brackets built-in code linting for JavaScript, and a very picky one.
Extensions :
Like most code editors, you can extend Brackets functionality with extensions of your choosing, or of your own crafting.

Now, by any means those are all of them, those are just the highlights of my choosing.

Available Options

With Brackets you have a variety of options, from spliting the view to creating your own themes, this section might also be looked as custumization options.

Split View

Editor(pane) in brackets can be more than one on the main view, the view can be splited into Two at the menu bar View › Vertical View or View › Horizontal View, the same thing can also be done in the top right corner on the sidebar.

Sidebar

The sidebar can be hidden or shown at View › Show/Hide Sidebar. Located in the left side of the main view, the sidebar provides informatios such as, the current open file, edited files(working files), unsaved files, working folders and if the view is splited the editors will also have splited lists of files inside the sidebar for each one of them.

In the sidebar, folders can be opened and created, files can be created, renamed, deleted, moved between editors(if more than one) and saved, anytime a file is open and edited, that file goes to Working Files and all of these happen within the Working Folder, which might be your entire project and its structure.

Quick View

This is also one of brackets helpful features, it works on Hover, in other words, when you place the cursor over a image for example, that image is shown in a tooltip way. This feature also works with colors in CSS files, and it can be desabled at View › Quick View on Hover.

Themes

Brackets can be even further custumized with themes, themes can change the colors that highlight your code, they can change fonts of the editor, and even the whole brackets layout can be changed with a simple change of theme.

Themes can be instaled or removed through the Extension Manager and they can be selected at View › Themes…, where you also have other customization options such as font size and family.

Live Preview Highlight

This is one of the options that i liked the most, when in Live Preview mode you get highlighted preview for the element you currenty have your cursor on, the feature is cursor-driven, it can be desabled by unchecking View › Live Preview Highlight.

More Options

In Brackets you can Increase, Decrease or Restore Font Size through the View Menu, your lines can be numbered ( View › Line Numbers ), blocks of code can be collapsed, cursor can be changed, overall Brackets as a wide variety of options to suit anyone's taste.

Options can be even further explored in the editor preferences files, read more about preferences from Brackets Wiki.

Working in Brackets

Personally, i found brackets very welcoming, the defaults are very good and is not likely that you will encounter any problems whatsoever.

With support for almost all modern languages, you'll start coding right after the installation.

The snapshot above shows the simplicity of brackets, and, a vair amount of earlier discused topics can be further clarified and identified. Now, for a closer look a fast installation of Brackets would give even more details about the environment.

Being a folder based coder editor, a good starting point is to open a project folder File › Open Folder… or from the sidebar. New files can also be added or created in one of those two methods.

Working with files

Editing files is very pleasant and with the features, editing becomes quite rich. Also, you have full control over your editor, it can be easily customized just by exploring through the available options, it won't take much to find what you like and feel comfortable with.

Now, working with files is straightfoward, all the files will be listed on the sidebar, Working Files on the top and Working Folder( the whole project ) will be right bellow. If the sidebar is somehow distracting, you can hide it or opt to select the no distraction mode at View › No Distractions.

In no distractions mode you will not be able to see the tree of files in your project , but, you can easily navigate through your files by accessing the Navigate menu, where you have a variety of helpful features such as Quick Open( open files with a simple search in the whole project ), Next or Previous Document by working order or by list order.

Depending on your preferences, the cursor mode can be changed between Insert or Overwrite mode at the bottom right corner, how your code is indented can be fully customized also at the bottom right corner and your options are between Tab Size( which the character width can be specified ) or in Spaces.

When you start to write your code you'll experience Code Hints and Auto Completion, they are active by default and can on be disabled in the default preferences file ( Debug › Open Preferences File ). A good amound of editing utilities can be found under the Edit menu, some basic and some not, but, they can definitely help you write faster and cleaner code.

If you write a lot of code in one file and you often get lost in your code, Brackets helps you find whatever you need to search/find or change in your project, you can quickly search( find only ) inside the working file ( Find › Find ) or inside the working folder ( project ) at Find › Find In Files, or, you can find and replace Find › Replace/Replace In Files.

As soon as you start to really get things done with Brackets you'll begin to use and feel the power of its features, when working with CSS and HTML documents in particular, Quick Edit will be helpful to quickly modify styles from within the html file, no need to go search for that one selector in one long css file, you can just place your cursor over a element, class or id that needs quick modification, select Navigate › Quick Edit and you'll be presented with a inline editor that searches for your selection in the stylesheets linked to the html file so you can make your changes. Optinally you can also use Navigate › Quick Jump To/Find Definition that can be used to perform a jump( go to ) within a file based on selectors, functions names, etc.

Working with Live Preview

The feature is basically a local server that serves your files statically, the preview is instantly, your browser as real-time updates while you write code, you can lunch it from within a html file by selecting File › Live Preview or by clicking the first icon ( lightning bolt ) from the panel at right corner, currently the feature is limited to only work with Chrome. Read more about this feature.

Working with Extract

Now, extract is a feature that might or might not be on your Brackets installation package, it depends if you choose the package that comes with it when downloading Brackets, but if not, you can install it from the Extension Manager.

It is certainly going to change your development process, but it is also a Creative Cloud service so you'll probably need a account to sign in with.

Once you open Extract on the right corner panel and sign in with your Adobe ID you can start by opening your design file, then you can start extrating informations. At the right panel of Extract there is a list of Layers just like in Photoshop, and at the main panel you get a preview of the file, when you select a element you get width and height and its absolute position, and by holding Shift when doing a selection you get the element's relative position to other element, but the extension becomes useful when used within a CSS document, in other words, when you select a element in extract you can go in your CSS and press Ctrl + Space to get its CSS propreties.

Installing Extensions

All extensions are managed through the Extension Manager at the right corner panel, whitin the Extension Manager you have three panels, Available, Themes and Installed, you can Search for extensions in the search bar or you can install them by dragging and dropping zip files or Install from URL links, you can disable or remove them at the installed panel.

There is a lot to choose from the Extension Manager, and in case of doubt you can click more info which will probably take you to a web page with informations about it, visit Brackets Extension Registry to browse through them all.

Shortcuts

The code editor is keyboard friendly and pretty much almost everything can be accessed with a shortcut, i think that shortcuts are very helpful as they give you a way of keeping your hands off of the touchpad and enhance the whole development process, so, see the list of shortcuts available in Brackests, take a look and if you don't grab much you can also see the shortcuts inside of Brackets on the submenus or Debug › Show Developer Tools › Settings › Shortcuts.

Final Insight

I use Brackets on a daily basis and i've yet to find some reason not to, its a powerful, simple, flexible and a keyboard friendly text editor that is nicely developed, and, open sourced with great contributions from its community.

There is a lot more to it than just this overview, but i hope this will get you started with a good base, although if you get to mess in Brackets you'll get to see that Brackets is very straightforward.
